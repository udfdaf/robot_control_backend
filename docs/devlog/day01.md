# Day 01 – 프로젝트 초기 세팅

## 목표
- 백엔드 프로젝트 초기 환경 구성
- 개발 및 실행이 가능한 기본 구조 완성

## 진행 내용
- Node.js(LTS) 및 npm 설치
- NestJS 기반 백엔드 프로젝트 생성
- 로컬 개발 서버 실행 확인
- Swagger를 이용한 API 문서화 환경 구성 (`/docs`)
- GitHub 저장소 연결 및 초기 커밋 진행

## 문제 및 해결
- 기존에 다른 GitHub 계정 자격 증명이 PC에 저장되어 있어 push 권한 오류 발생
  - Windows 자격 증명 관리자를 통해 기존 GitHub 인증 정보 삭제 후 재인증하여 해결

## 결과
- 로컬에서 NestJS 서버 정상 실행
- Swagger를 통한 API 문서 접근 가능
- 기능 개발을 바로 시작할 수 있는 상태로 프로젝트 준비 완료
